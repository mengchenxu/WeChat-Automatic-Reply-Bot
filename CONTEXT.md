# 群聊 AI 机器人 — 领域词汇表

## 群成员 (Group Member)

微信群中的一个人。bot 通过 wxid 唯一标识每个群成员。

### 名字槽 (Name Slots)

每个群成员有三种名字：

| 名字 | 英文 | 定义 | 来源 | 可变性 |
|------|------|------|------|--------|
| **基准名** | Mention Name | 微信 @mention 能搜到的群昵称，bot @人时用的权威名字 | 群成员 API 的 `displayName` | 群主/本人改群昵称时变化，bot 每 30 分钟检查一次 |
| **外号** | Alias | 群里人实际叫的名字，bot 自己学的。基准名自动加入外号列表 | LLM 提取 + 基准名变更时旧名迁移 | 只增不减 |
| **消息名** | Preferred Name | 消息流中最近出现的原始显示名，可能被 wxid 污染 | WeFlow 消息的 `display_name` | 每条消息都可能变 |

### 名字解析规则 (Name Resolution)

bot 需要把消息中出现的任何名字映射到具体的人。查找优先级：

1. **基准名** — 精确/子串匹配 `mention_name`
2. **外号** — 匹配 `aliases` 中的任意一个
3. **消息名** — 匹配 `preferred_name`
4. **历史名** — 匹配 `display_names` 历史
5. **wxid** — 最后手段

### 基准名变更 (Mention Name Rotation)

当群昵称被修改时（bot 在定时同步中发现基准名变化）：

1. 旧基准名 → 加入 `aliases`（保证旧名字仍然能搜到）
2. 新基准名 → 写入 `mention_name`
3. 新基准名 → 同时加入 `aliases`

## 群情景记忆 (Group Episodic Memory)

bot 对"群里发生过什么"的记录。每条记忆包含：内容、分类（事件/决定/笑话/话题转折）、参与者名字、检索关键词、重要度。

记忆通过关键词匹配 + 时间衰减检索，定期由 LLM 从最近消息中提取。

## 用户事实 (User Fact)

bot 对一个人的认知。以键值对存储（如 `立场=支持猎鹰`），带来源和置信度。可被纠正信号（"我不是xxx"）触发修正。

## 群上下文 (Group Context)

整个群的持久化摘要，2-4 句话概括群成员特征、当前话题和共识。定期由 LLM 更新。
